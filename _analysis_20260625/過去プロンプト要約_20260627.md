# 過去プロンプト要約（語根分解アプリ徹底ブラッシュアップ）

作成日: 2026-06-27 / 対象: 3アプリ（日本語版・中国語版・韓国語版）の語根分解・注釈ルビ・漢字化の精度向上作業

---

## 0. 全体の目標（ユーザーの一貫した意図）

京大エス研HTMLコーパス群と**同水準（限りなく100%に近い）の語根分解精度**を、
アプリの分解システムを最大限活用し、**法則性を見出した汎用性の高い調整**で再現する。
3つの達成目標：

1. **語根分解** — ほぼ100%の精度。
2. **注釈ルビ** — 日中韓語訳でエスペラント学習に適切な注釈ルビを振る。
3. **漢字化** — 偽分解（国際語の語源的分解など）にもしっかり漢字を割り当てる。

最重要の繰り返し指示：**「マスター語根分解辞書、マスター漢字割り当て文書をしっかり使う」**。

---

## 1. 語根分解の精度向上

- 「これらのHTML文書群（vere_aux_fantazie / Esperanto_meznivela 等、京大エス研GitHub収納）と
  同水準の語根分解精度が再現できれば最高」
- 「代名詞・所有形も参照どおり分解する（nia→ni/a 等。同綴り衝突を避けつつ）」
- 「もう一方の vere_aux_fantazie.html でもベンチ測定」
- **設計思想（ユーザー本人の説明）**:
  「一つの文字の並びに対しては一通りの分解しかありえない。基本的に**汎用性の高い語への語根分解が優先**される。
  両立させるために**語尾やスペースを含めてみたり**する」
  → 汎用語優先＋語尾/スペースで両立、という分解設計。

## 2. 注釈ルビ

- 「注釈ルビについては、エスペラント学習に適切な注釈ルビを**日中韓語訳**で振ることが目標」
- 「注釈自体は [gold] や [HTML修正ガイド] の方が信頼できる」
- 「漢字割り当てはマスター準拠ですべき。**逆に注釈についてはコーパスHTMLの方がいい**かな」
  → **ルビ=コーパス準拠、漢字=マスター/gold準拠**、という二系統の使い分けに決定。
- 「日中韓語注釈についてはどのようになっていますか？マスターを使うとすると
  そこまで簡単な話ではないように思うが、きちんと作れているのですか？」（要確認の指摘）

## 3. 漢字化（偽分解への漢字割り当て）

- 「**偽分解とかにもしっかり漢字を割り当てて行った方がいい**」
- 「esperanto は割り当てる漢字があったはず。。。**望在**」
- 「漢字化は元となる最新のリストを遵守してください。**esper に望が、ant には在**が割り当てられているはず」
  → esperanto = esper(望)+ant(在) の偽分解で漢字を割り当てる（masterの分解に従う）。

## 4. マスター遵守・発明の厳禁（強い叱責あり）

- 「**マスター語根分解辞書、マスター漢字割り当て文書はフルに活用できているのですか？**」
- MITAIO_OVERRIDE（german→徳, rus→俄 等の**勝手な漢字割当**）に対し：
  **「👆何勝手なことをしているのですか？？」**
- 「私の発明的な override を**完全撤回**し、マスターを遵守します。👆お願いいたします！」
  → 発明・憶測による割り当ては**厳禁**。常にマスター文書に存在する割当のみを使う。

## 5. アプリ機能・運用方針

- 「今アプリの機能として、要らなくなってしまっているような機能はありますか？」（問い）
- それへの回答方針として：
  1. **「消さずに、json生成機能から磨いて欲しい」**（機能削除ではなく生成機能の高精度化）
  2. streamlit cloud 用のファイルは**必要**（消さない）
  3. 生成機能の高精度化を**継続して欲しい**
  4–5. 必要（保持）

## 6. バックアップ

- 「**外部ssdにバックアップしっかりとってくれ！！**」
- 辞書・漢字割当・アプリを定期的に Downloads + 外部SSD へバックアップ（三重化方針）。

## 7. 触れてはいけないマスター原本（読み取り専用）

- `\\wsl.localhost\Ubuntu\home\y\エスペラント辞書徹底語根分解_20260619`
- `D:\…\漢字化・語彙資料\エスペラント語根＿漢字割り当て＿20260621`
- **「こちらからは編集しないでね」** → 参照のみ。アプリ側で複製・反映する。

## 8. 確定した設計上の結論（作業を通じての合意）

| 項目 | 正本（マスター） | 適用先 |
|---|---|---|
| 語根分解 | WSL 学習者版 `…_学習者版_utf8_20260416.txt`（=gold, 62283行） | 分解の骨格 |
| 注釈ルビ | **京大エス研コーパスHTML**（多数派の分解に追従） | ルビの分かち書き |
| 漢字割り当て | `_kanji_map_master.tsv`（per-root）+ `漢字注入_学習者版_20260620.txt`（per-word） | 漢字化（偽分解含む） |
| 日中韓注釈 | `…_日中韓注釈版_ドラフト.txt`（8.8MB, per-word 日中韓3言語） | word_anno訳語 |

- **gold(国際語を語源分解) vs corpus(一体保持) の対立** → **ルビ=corpus / 漢字=gold** で両立（KANJI_DECOMPOSE 機構）。
- 3アプリは**絶対に commit/push しない**。VCするなら完全別リポジトリ（esperanto-radiko-cjk-annotator）。
